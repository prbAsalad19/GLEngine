# Security Policy

## Supported Versions

GLEngine is currently an experimental project and does not maintain long-term supported versions.

Security issues will be addressed on the latest version of the codebase.

## Reporting a Vulnerability

If you discover a security vulnerability, please **do not open a public issue immediately**.

Instead, report the vulnerability privately by contacting the maintainer through GitHub.

When reporting a vulnerability, please include:

- A clear description of the issue
- Steps to reproduce the problem
- Potential impact if known
- Any possible mitigation or fix suggestions

## Response Process

Once a vulnerability is reported:

1. The issue will be reviewed by the maintainer.
2. A fix will be developed if necessary.
3. The vulnerability may be disclosed publicly after a patch is available.

## Scope

This policy applies to vulnerabilities affecting the GLEngine codebase.

General bugs or feature requests should be reported through the GitHub issue tracker.
