# Contributing to GLEngine

Thank you for your interest in contributing to GLEngine!

## Reporting Bugs

If you find a bug, please open a bug report issue using the "Bug report" template.

Include:

- A description of the problem
- Steps to reproduce
- Expected vs actual behavior
- Environment (OS, compiler, engine version)

## Suggesting Features

Feature requests are welcome. Open an issue using the "Feature request" template.

Include:

- What the feature is
- Why it would be useful
- Optional: How you would implement it

## Pull Requests

When submitting a pull request:

1. Keep changes focused and minimal
2. Ensure the code compiles and runs
3. Update documentation if needed
4. Follow existing coding style

## Code Style

- Classes, engine-level functions and variables: CamelCase
- Math/utility functions: snake_case
- Indentation: 4 spaces
- Keep functions short and modular
